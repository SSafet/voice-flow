5 · Rules

  • Destination is data, not UI state: every capture stores one destination — pasted into app · kept · → assistant · → session N — shown only as quiet meta text next to the timestamp, never as a pill/badge. The Inbox filters are views over that field; ticket #2’s inbox is the Kept filter, never a fifth tab.
  • Kept items are unread until revisited: the amber glow dot; “✓ seen” (or opening the item) clears it. The Inbox tab count = unrevisited kept items.
  • Agents rail ≡ pill picker: same sessions, same numbers, same unread rings, assistant pinned as A. One mental model in both surfaces; ghosts stay readable until their stack is cleared (#14).
  • Asks are brighter: a push that wants an answer carries the amber border + inline answer box — the panel twin of the pill’s ask rule.
  • Speech is a verb, not a place: the drawer + per-row 🔊 replace the tab. Audio still never auto-plays.
  • Sync boundary (#7): Inbox = captures up, Agents = messages down — the two streams a future mobile client syncs, unchanged.
